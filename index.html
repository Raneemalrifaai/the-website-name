How to publish your website
on GitHub Pages

Step-by-step guide to uploading your HTML files and making them live on the internet — for free, in under 5 minutes.

Part 1 — Create your GitHub account & repository
1

Go to GitHub and create a free account

Open your browser and go to github.com — click Sign up and create a free account. Your username will become part of your website address, so choose it carefully.

Your website address will be: https://your-username.github.io/your-repo-name/
github.com
github.com → click Sign up in the top right corner → enter your email, password, and username → verify your account
2

Create a new repository

A repository is like a folder on GitHub that holds all your files. This is where you will upload your HTML files.

+ button (top right)
→
New repository
Repository name: type a short name, e.g. kanaleneiland — no spaces, use dashes

Description: optional, e.g. "WHO Healthy Cities Analysis"

Visibility: choose Public (required for free GitHub Pages)

Leave everything else as default — do not tick "Add a README file" yet

Then click the green Create repository button
Important: The repository must be Public — Private repositories cannot use the free GitHub Pages hosting.
Part 2 — Upload your HTML files
3

Upload your files to the repository

After creating the repository you will see an empty page. Now you need to upload all your HTML files.

uploading an existing file
— click the link on the empty repo page
Step 1: On the empty repository page, click the link that says "uploading an existing file"

Step 2: Drag and drop ALL your .html files at once into the upload area — or click "choose your files" to browse

Step 3: Wait for all files to finish uploading (you will see a green tick next to each one)

Step 4: Scroll down to the Commit changes section — leave the message as "Add files via upload"

Step 5: Click the green Commit changes button
Upload all files at the same time — index.html, bubble_map.html, 6p_criteria.html, cascade_map.html, intervention_sequence.html, full_intervention.html, location_simulator.html, sources.html
The file named index.html is your homepage. When someone visits your website address, GitHub automatically opens index.html first. Do not rename it.

Part 5 — How HTML files become page titles
7

How the <title> tag sets the browser tab name

Every HTML file has a title tag in the <head> section. This is what appears in the browser tab, in Google search results, and when someone bookmarks the page.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Kanaleneiland — Healthy Cities Analysis</title> ← this is the browser tab name </head> <body> ... your page content goes here ... </body> </html>
Browser tab shows the <title> tag content
Kanaleneiland — Healthy Cities Analysis ×
What you write in <title></title>
appears in the browser tab above
Each of your HTML files has its own title tag — that is why each page has a different tab name when you navigate between them.

bubble_map.html → title: "Kanaleneiland — WHO Healthy City Relationship Map"

Part 6 — How index.html links to all your other pages
8

Links between pages use the file name

In index.html, each card that opens another page uses a simple <a href> link pointing to the filename. That is how clicking a card on the homepage opens the right page.

<!-- This is how a card link works in index.html --> <a href="bubble_map.html"> ← the filename you uploaded <div class="card-num">01</div> <div class="card-body"> <h2>Relationship Bubble Map</h2> ← card heading shown to user <p>Description text here</p> </div> </a> <!-- href="bubble_map.html" must exactly match the filename you uploaded -->
The most important rule: the text inside href=" " must be spelled exactly the same as the filename you uploaded — including lowercase letters and underscores. If your file is called bubble_map.html then href must say "bubble_map.html" — not "Bubble_Map.html" or "bubblemap.html".
Part 7 — Updating your website after changes
9

How to update a file after you have made changes

When you update an HTML file and want the website to reflect the change, just upload the new version to GitHub.

1
Go to your repository on github.com
2
Click the file you want to replace (e.g. full_intervention.html)
3
Click the pencil icon (Edit) → then at the top right of the editor click the three dots ··· → Delete file — or simply click the upload icon to replace it
4
Alternatively: on the main Code tab, click Add file → Upload files → upload your new version → GitHub automatically replaces the old one if the filename is the same
5
Click Commit changes — your website updates in about 30–60 seconds
Quick tip: After committing, wait 30 seconds then do a hard refresh in your browser (Ctrl+Shift+R on Windows, Cmd+Shift+R on Mac) to see the updated version — your browser may have cached the old page.
